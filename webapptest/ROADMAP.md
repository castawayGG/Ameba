# Ameba — Оставшиеся ключевые задачи и возможности

Документ составлен на основе анализа текущих pull request'ов и истории задач.

---

## ✅ Реализовано

- Управление аккаунтами Telegram (статус, сессии, 2FA, теги, пул)
- Управление прокси (bulk, авто-загрузка, ротация мобильных)
- Кампании рассылок (голос, кружки, инвайты, шаблоны)
- Парсинг: гео, комментаторы каналов, lookalike, scrub
- AI-фарминг: генератор комментариев (OpenAI/Claude), генератор личностей, реакции
- Антидетект (fingerprint/профили)
- Cooldown-менеджер
- Спинтакс-шаблоны
- Quick Reply + сниппеты
- Контакты и диалоги аккаунта (парсинг, рассылка по контактам)
- Команда (задачи, объявления, общие шаблоны)
- Воронка конверсии (кампания: отправлено → ответы → клики)
- ROI/Дашборд операторов
- Breadcrumbs, глобальный поиск (Ctrl+K), scroll-map
- Макросы + история bulk-операций
- Облачный бэкап (Yandex Disk / Google Drive)
- CSV-импорт аккаунтов (add/replace/merge/delete режимы)
- Гранулярные права доступа (JSON permissions)
- White-label брендинг (логотип, цвет, название)
- Переключатель языка RU/EN
- Telegram-алерты (бан, падение прокси, входящие, завершение кампании) — индивидуальные настройки на пользователя
- Вебхуки (HMAC, события, логи доставки)
- API-ключи для сторонних интеграций
- Клоакинг (User-Agent/IP)
- DNS-ротация через Cloudflare
- Криптобот (баланс)
- Консоль логов в реальном времени (AJAX polling)
- Отчёты (HTML/PDF, Excel, отправка в Telegram)
- Инструмент конвертации сессий
- A/B-тестирование
- Мониторинг сервера
- Аудит-логи, история действий администраторов

---

## 🔧 Реализовано (этот PR)

- **Изоляция данных по владельцу** — аккаунты видны только своему владельцу (роль `admin`/`editor`); superadmin видит все
- **Авто-импорт из внешних API** (`/admin/number_import`) — загрузка номеров/аккаунтов из внешних HTTP-источников (JSON API, CSV URL)

## ⏳ Оставшиеся ключевые задачи

### 1. Полная мультитенантность (SaaS-режим)
- [ ] Назначить `owner_id` прокси, кампаниям, диалогам
- [ ] Изолировать видимость прокси между командами
- [ ] Страница регистрации нового пользователя с личной «базой» (аккаунты, прокси, кампании)
- [ ] Ограничение квот на команду/пользователя (max_accounts, max_campaigns и т.д.)

### 2. Экспорт в Google Sheets
- [ ] Интеграция через `gspread` (OAuth / service account)
- [ ] Кнопка «Экспорт в Google Sheets» для контактов/диалогов и аккаунтов
- [ ] Настройки подключения (credentials JSON) в `/admin/settings`

### 3. Автоматическое обнаружение лидов (Lead Detection)
- [ ] Сканирование входящих сообщений на ключевые слова (интерес к продукту)
- [ ] Автоматическая пометка диалога как «горячий лид»
- [ ] Уведомление оператору в реальном времени
- [ ] Интеграция с CRM (webhook при появлении лида)

### 4. Планировщик кампаний
- [ ] Запуск кампании по расписанию (cron-выражение / дата/время)
- [ ] Пауза/возобновление кампании в расписании
- [ ] Лимит по часам/дням на аккаунт
- [ ] Визуальный календарь кампаний в dashboard

### 5. Расширенный A/B-тест
- [ ] Автоматическое определение победителя (по reply rate / click rate)
- [ ] Распределение трафика в процентах (50/50, 70/30 и т.д.)
- [ ] История результатов A/B-тестов

### 6. Интеграция CRM и внешних платформ
- [ ] Синхронизация контактов с amoCRM / HubSpot через API
- [ ] Двусторонний webhook (получение лидов из внешних CRM)
- [ ] Интеграция с сервисами коротких ссылок (bit.ly и т.д.) для TrackedLink

### 7. Аналитика и отчётность
- [ ] Фильтрация дашборда по дате, аккаунту, прокси, кампании
- [ ] Тепловая карта активности по часам/дням
- [ ] Экспорт отчётов в Google Sheets
- [ ] E-mail рассылка отчётов (SMTP интеграция)

### 8. Улучшения безопасности
- [ ] 2FA для всех пользователей (принудительно по ролям)
- [ ] IP-allowlist на уровне пользователя (не только глобальный)
- [ ] История входов пользователя с гео-данными
- [ ] Автоматическая блокировка при подозрительной активности

### 9. Прочие UX-улучшения
- [ ] Тёмная/светлая тема по выбору пользователя (сохранение в профиле)
- [ ] Уведомления в реальном времени через WebSocket (не только polling)
- [ ] Мобильный вид (responsive) для ключевых страниц
- [ ] Undo для критических операций (удаление аккаунтов/прокси)
